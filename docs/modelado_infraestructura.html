📂 6. Artefactos Desplegados (Archivos Clave)
Artefacto Ruta Responsabilidad Capa
main.py src/api/main.py Aplicación FastAPI: endpoints REST, webhooks WhatsApp y E-Commerce, archivos estáticos API Gateway
mas_orchestrator.py src/agents/mas_orchestrator.py Construcción y compilación del StateGraph de LangGraph con los 4 agentes y aristas condicionales Framework MAS
store_agent_node.py src/agents/store_agent_node.py Agente de Tienda: interfaz conversacional WhatsApp, gestión de ventas/ingresos/mermas Agentes
coordinator_agent_node.py src/agents/coordinator_agent_node.py Agente Coordinador: consolida transacciones en el kárdex, control de concurrencia Agentes
sync_agent_node.py src/agents/sync_agent_node.py Agente de Sincronización: procesa órdenes web (inbound) y sincroniza stock (outbound) Agentes
alert_agent_node.py src/agents/alert_agent_node.py Agente de Alertas: notificaciones proactivas de stock bajo y anomalías de mermas Agentes
whatsapp_gateway.py src/gateway/whatsapp_gateway.py Integración con Meta Cloud API: envío/recepción de mensajes WhatsApp Gateway
message_router.py src/gateway/message_router.py Enrutador: parsea webhooks WhatsApp, invoca el grafo LangGraph, envía respuesta Gateway
models.py src/database/models.py Modelos SQLAlchemy: Product, ProductVariant, Transaction, AgentLog, ChatSession Datos
state.py src/agents/state.py Definición de MASState (TypedDict) y función create_message (protocolo FIPA-ACL) Estado MAS
ℹ️ Nota Técnica: En el prototipo actual, todos los componentes (FastAPI, LangGraph MAS, Gateway WhatsApp y la BD SQLite) se ejecutan dentro de un único proceso Python en Railway. La comunicación entre agentes es por invocación en memoria (mismo hilo), no por red. En una arquitectura de producción escalada, los agentes podrían distribuirse como microservicios independientes con comunicación vía colas de mensajes (Redis/RabbitMQ).
